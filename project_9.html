왜 만들었나

V1 · Python exe

스크린샷 → OCR → 스프레드시트 자동 입력. 동작은 했지만 수동 조작 필요, 다른 사람이 쓰기 어려움.

V2 · 카카오톡 챗봇

AI 챗봇으로 개선했으나 서버용 PC 24시간 가동 필요 → 발열·소음 문제로 3일 만에 종료.

V3 · Flutter 앱

서버리스 클라우드 아키텍처. 별도 서버 없이 스마트폰에서 바로 사용 가능.

문제 해결 · 핵심 사례

iOS 무한 로딩 문제

증상
iPhone에서 사진 인식 후 로딩이 끝나지 않음. Firestore에는 데이터가 정상 저장됨.
원인
Firestore 트랜잭션 성공 후 Storage 업로드 단계에서 iOS가 멈춤.
1차

임시 해결: 타임아웃 추가

Storage 업로드에 timeout 적용으로 무한 대기 차단

발견

Storage 사용량 80%가 출결 사진

OCR 후 재사용 가치 없는 사진이 비용의 대부분을 차지

최종

출결 사진 업로드 자체 제거

OCR 입력으로만 사용, 서버 저장 중단

문제 해결 · 보안

스스로 발견한 보안 취약점

정상 사용자 흐름은 유지하면서 악용 가능성은 차단

채팅 미리보기 위조
lastMessage·lastMessageTime을 클라이언트가 직접 갱신하면 채팅방 미리보기 조작 가능
→ Cloud Functions으로 갱신 권한 이전
시스템 메시지 위조
isSystem:true 메시지를 클라이언트가 임의 생성 가능 → 가짜 퇴장·삭제 안내
→ 시스템 메시지 생성을 Functions 전용으로 제한
강제 그룹 참여
그룹 생성 시 participants에 타인을 바로 추가 → 원치 않는 그룹에 강제 입장
→ chat_invites 문서 기반 초대 수락 흐름으로 전환
기본 학급방 권한
학생이 먼저 학급방 생성 시 구성원 임의 설정 가능
→ ensureDefaultGroupChats (admin SDK 전용) 으로 이전
개인정보 노출
비참여자도 학급방 문서의 participantNames 조회 가능
→ Firestore rules 참여자 전용으로 축소
출결 중복 저장
다른 기기 동시 업로드 시 중복 문서 생성 가능
→ Deterministic ID + Firestore 트랜잭션으로 원자적 처리

생기부 요약

Flutter와 Firebase를 활용해 도제 현장실습 출결을 자동화하는 앱을 기획·개발하였다. OCR로 출결 시간을 자동 추출하고, Firestore 보안 규칙으로 역할별 접근 권한을 분리하며, Cloud Functions으로 서버 자동화를 구현하였다.
