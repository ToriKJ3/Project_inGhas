왜 만들었나

기존 방식의 문제
학생이 교무실에 직접 찾아가야 한다
선생님이 자리에 계신지, 수업 중인지, 출장인지 미리 알 방법이 없다. 찾아갔다가 없으면 다시 오거나 다른 선생님께 물어봐야 한다.
교사 입장의 문제
반복적인 문의로 업무 흐름 단절
수업·회의 중에도 학생 문의가 이어질 수 있다. 교무실 내부에서도 특정 선생님을 찾는 반복이 발생한다.
해결 방향
교무실 자리 + 상태를 웹으로 실시간 공개
재실 상태인 선생님에게만 호출 가능. 교무실 디스플레이에서 음성 안내까지.
탐구 목적 5가지
1

자리 배치 시각화

교무실 좌석 배치를 웹 화면에 카드로 표현

2

실시간 상태 공유

재실·수업중·회의·출장 등을 실시간으로 공유

3

조건부 학생 호출

재실 상태일 때만 호출 가능 → 오작동 최소화

4

호출 현황판 + 음성 안내

교무실 디스플레이에서 실시간 수신·관리

5

관리자 편집 기능

코드 수정 없이 자리 배치·이메일 매핑 관리

설계 결정과 해결

문제
호출창 열어둔 상태에서 선생님이 자리를 비움
해결
전송 직전 상태 재확인. 재실 아니면 호출 차단.
실시간 시스템에서 발생할 수 있는 상태 불일치 문제를 이중 검증으로 보완
문제
TTS API 키를 클라이언트에서 직접 호출하면 노출
해결
Cloud Functions 프록시. API 키는 Functions Secret으로 관리.
클라이언트 노출 없이 외부 TTS API 사용
문제
매일 자동으로 상태와 호출을 정리해야 함
해결
21시 자동 마감 로직. 호출 삭제 + 상태 → 퇴근. 세션은 유지.
다음 날 재실만 설정하면 바로 사용 가능. 재로그인 불필요.

한계 및 개선 방향

솔직한 한계 분석

현재 한계
①PIN 검증이 클라이언트 중심 — 강력한 보안 인증이 아닌 오작동 방지 장치. 서버 측 검증 강화 필요.
②Security Rules 부분적으로 열린 구조 — 호출·상태 접근성 확보 위해 일부 경로 개방. 실 배포 시 역할별 권한 세분화 필요.
③학생 인증 없음 — 한글 이름 입력만으로 호출 가능. 학번 인증·호출 횟수 제한 등 추가 가능.
④TTS 외부 API 의존 — Supertone 장애 시 음성 안내 불가. 브라우저 기본 TTS로 폴백 보완 중.
⑤네트워크 의존성 — 웹 기반이므로 오프라인 또는 불안정 환경에서 실시간 동기화 지연 가능.
